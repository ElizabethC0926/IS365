文件 Function（职责） Input Output User
tax_params.jsonP0 按年份存储所有税法数字：Grundfreibetrag、税阶上限、AfA率、15%阈值、66%阈值、各州契税率。税法变化只改此文件，代码零修改 人工维护（来源 BMF 年度公告） 被 tax_engine.py 加载，提供当年参数 系统内部
config.jsonP0 存储不随税法变化的业务常量：仿真年限默认值、采样数量N、FLAG定义名称、输出字段列表 人工维护 被所有 .py 文件读取 系统内部
tax_engine.pyP0 德国税法计算函数库：累进税率（含 Splittingtarif）、AfA折旧、15%资本化规则、66%租金规则、§23投机税、Werbungskosten 汇总。从 tax_params.json 读参数，代码零硬编码 各函数接收单一数值参数（不依赖整个 PropertyCase），+ year: int 各函数返回 dict，含计算结果 + 所用参数 + 法条引用（为 FLAG 和 Decision Log 准备） 系统内部
finance_engine.pyP0 融资计算：贷款摊销表（逐年利息/本金/余额）、Sondertilgung 效果、再融资节点建模、年度现金流净值（租金收入 - 贷款支出 - 维修 - 税） principal, rate, years, sondertilgung_pct, refi: bool list[YearlyLoanState]：每年的利息额、本金还款额、剩余贷款余额 系统内部
property_model.pyP0 PropertyCase 数据类：持有一套策略组合的全部参数快照。暴露 step_year(year) 方法推进年份状态，暴露 exit(year) 方法计算退出结果 验证后的策略参数 dict（来自 sampler 或用户输入） PropertyCase 对象，供 simulator.py 逐年驱动 系统内部
flag_system.pyP0 检测4个 FLAG：15%规则触发、租金过低、税务额度浪费、现金流赤字。每个 FLAG 记录是否触发 + 触发年份。这是用户 filter 和风险提示的数据来源 list[YearlySnapshot] + PropertyCase FlagResult dataclass：{FLAG_名称: {triggered: bool, year: int | null}}，直接序列化为表格列 用户（作为结果表的列，可 filter）
simulator.pyP0 单策略完整生命周期仿真：驱动 PropertyCase 逐年 step，每年调用 tax_engine + finance_engine，汇总逐年快照，计算退出结果，调用 flag_system 生成 FLAG。这是层一的总装文件 PropertyCase 对象 SimulationResult：{yearly_snapshots, exit_result, flags, cashflow_series}。纯函数，无全局状态，支持并行 系统内部
validators.pyP1 参数入口约束校验：LTV范围、rental_ratio与usage一致性、land_ratio合法区间、holding_years上下限。返回全部错误列表（非 fail-fast），在参数进入仿真前拦截 用户输入的参数 dict ValidationResult：{valid: bool, errors: list[str]} 系统内部（错误信息透传给用户）
sampler.pyP1 策略空间穷举/采样：按 V1.0 四个维度的离散分级生成所有合法参数组合。强制包含碰撞测试组（触发15%规则）和退出时机对比组（第9/11年）。每个组合带 group_tag 用户固定参数（房价、收入、地点）+ config.json 中的维度定义 list[dict]：每个 dict 是一套完整策略参数，含 group_tag，已通过 validators 验证 系统内部
irr_calculator.pyP1 IRR / NPV 数值求解。接收纯现金流序列，与业务逻辑完全解耦。IRR 是结果表的一列，用户可用来 filter（"只看IRR > 4%的方案"） list[float]（现金流时间序列，首项为负的初始投入） {irr: float | nan, npv: float}。无解时返回 nan，不抛异常 用户（结果表的一列，可 filter）
output_formatter.pyP1 把 SimulationResult 扁平化为结果表的一行。每列对应一个用户可见维度：输入参数列、逐年现金流列、FLAG列、IRR列、退出净收益列。这是层一面向用户的唯一出口 SimulationResult + 原始参数 dict dict（一行结果表数据），字段与 schema.json 定义严格一致 系统内部（输出被 main.py 汇总成完整表格）
schema.jsonP2 结果表的字段规范：每列的名称、类型、含义、是否对用户可见、是否可 filter。是 simulator 输出与前端 filter 之间的契约文件 人工维护（随功能迭代更新） 被 output_formatter.py 验证输出格式，被前端读取生成 filter UI 间接（定义用户能看到什么列）
层二 · DL 优化模型 论文核心贡献，学习策略空间的结构 交付标准：比随机采样更高效地覆盖有价值的策略组合
文件 Function（职责） Input Output User
env.pyP0 Gym 风格 RL 环境：将 simulator.py 包装为 reset() / step(action) 接口。定义观测空间（当前策略参数状态）和动作空间（调整哪个维度、调整多少） 用户固定参数（房价等） observation / reward / done / info（每步交互） 系统内部
reward.pyP0 Reward 函数（论文最核心文件）：主奖励来自现金流质量，FLAG 触发作为惩罚项，每个惩罚项对应一条税法条款。Agent 通过最大化 reward 自然学会规避非法结构 SimulationResult + FlagResult float（scalar reward）+ 各分项分解 dict（供 Decision Log 记录） 系统内部
policy_net.pyP1 Policy Network 架构定义（具体选型是你的论文贡献：MLP / Transformer / 其他）。接收策略参数的向量化表示，输出下一步动作的概率分布 observation tensor action distribution + value estimate 系统内部
train.pyP1 训练循环：Agent 与 env 交互、收集经验、更新 policy_net 权重。记录训练曲线（reward 收敛情况）。产出训练好的模型权重 env + policy_net + 超参数配置 model.pt（模型权重）+ training_log.json 系统内部（研究者）
evaluate.pyP2 评估训练后的 Agent：与随机采样 baseline 对比，衡量 Agent 是否能用更少的仿真次数覆盖更多高质量策略。论文实验章节的数据来源 model.pt + 测试用固定参数集 evaluation_report.json（含对比指标，写入论文） 系统内部（研究者）

文件 Function（职责） Input Output User
main.pyP0 系统入口和批量编排：接收用户固定参数，调用 sampler 生成所有策略组合，并行调用 simulator 跑完所有仿真，调用 output_formatter 生成每行，汇总成完整结果表写入输出 用户固定参数（CLI 或 API 接口） results_table.csv + results_table.json（完整策略结果表，每行一个策略） 用户（最终消费这张表）

decision_log.pyP1 为每个策略记录关键决策节点的推理链：哪一年触发了什么规则、对现金流产生了多少影响、对应哪条法条。用户展开某个策略时看到的"为什么"内容 SimulationResult + FlagResult + rule_registry.json list[DecisionEntry]：每个关键年份一条记录，含规则触发描述和量化影响 用户（展开策略详情时可见）
scenarios.jsonP2 PO 预定义的对比场景模板：如"第9年 vs 第11年退出"、"高LTV vs 低LTV"。用户可一键加载这些场景作为 filter 的起点，无需手动设置参数 PO 人工定义 被 main.py 读取，注入 sampler 生成对应策略组合 用户（预设 filter 场景）
